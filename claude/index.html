系統流程架構

產生 可控九宮格與 Ground Truth
干擾 雜訊、遮罩、偏移、污染色塊
捕捉 中央 80% ROI 或偏移 ROI
辨識 平均、投票、綜合加權
決策 權重比例產生最後答案
紀錄 capture_log 與 weighted log
校正 交給 AI 分析錯誤與調權重

資料紀錄設計

Sheet 1: capture_log 每次掃描都寫入，記錄格子、演算法、干擾條件、辨識結果與是否命中。
Sheet 2: weighted_decision_log 只有綜合加權模式寫入，保存三個基礎演算法結果、權重、分數與最後決策。

AI 回饋校正用途

  • 分析哪種干擾最容易讓辨識失準。
  • 比較中央平均、整格平均、主要投票在不同條件下的可靠度。
  • 根據 weighted_decision_log 建議下一輪權重比例。
  • 延伸到產線燈號、監控畫面、OCR 欄位與 Vision Agent 測試。

教學說明專區

這個專區把干擾設計、Ground Truth 定義、雙紀錄表與 AI 協作方式放在同一個 `index.html`，適合直接推到 GitHub Pages。

操作案件對應

1. 開始同時啟動產生器與掃描器，形成連續觀測資料。
2. 結束停止定時任務，但保留目前畫面與紀錄，方便觀察結果。
3. 產生一次只刷新 Ground Truth，可用來控制單一測試樣本。
4. 掃描一次只做一次辨識，適合比對不同演算法或干擾條件。
5. 改變版面配色提供標準、實驗綠、活力橘三種白底展示風格。
＋ / － 視窗縮放放大或縮小整個介面，用來示範 Computer Use 操作視野。

鍵盤熱鍵

S：開始start，啟動連續流程。
E：結束end，停止連續流程。
G：產生一次generate，刷新 Ground Truth。
C：掃描一次capture，執行一次辨識。
T：切換配色標準、實驗綠、活力橘輪流切換。
+ / -：整體縮放放大或縮小整個操作介面。

這些熱鍵讓人類、鍵盤操作者與 AI Computer Use 都能用同一套明確任務控制 benchmark。

這個版本不是把工具重寫一次，而是把它「整理成可以丟給 AI 看的形狀」。學生不需要會工程，只要會打開 HTML、會問 AI 問題，就能跟著這份範例一起做。

學生操作八步驟

Step 1直接用瀏覽器打開 index.html，不需要任何安裝。
Step 2按「產生一次」，先看九宮格出現了什麼顏色。
Step 3按「掃描一次」，看下方表格寫進第一筆紀錄。
Step 4切換不同演算法（中央／整格／投票／綜合），重新掃描。
Step 5加入干擾（白霧、黑幕、反色塊…），再掃描一次。
Step 6切到「分析圖表」，看哪個演算法在哪種干擾下表現最差。
Step 7按「匯出 JSON」，把這份完整紀錄存下來。
Step 8把 JSON 或整個 HTML 丟給 Gemini／Claude／ChatGPT，請它解釋。

把 HTML 丟給 AI 時可用的 Prompt 範例

請用國中生也聽得懂的話，解釋這份 index.html 的資料流。
請列出每個 SECTION（用 /* === SECTION:XXX === */ 標記的那些）各自在做什麼。
請幫我新增一種叫 rainbowStripe 的干擾，並說明你改了哪一段、為什麼那樣改。
請幫我把 palette 換成柔和粉色系，但保持 9 種顏色都區分得開。
請幫我新增一個分析圖表，顯示每個格子位置（row,col）的平均準確率。
請幫我把 Mode 2 數字九宮格做一個最小可運作的雛形。
請幫我新增一個熱鍵 R，按下時隨機切一種干擾。
我把匯出的 JSON 貼給你，請告訴我這次測試中哪個演算法最常把藍色判成紫色。

小訣竅：先請 AI「只解釋，不要改」一次，確認它真的讀懂了，再請它動手改。

模仿練習任務

練習 1改 palette 顏色 — 換成柔和粉色系或公司品牌色。
練習 2新增一種干擾 — 例如「條紋遮罩」或「噪點變多」。
練習 3改 ROI 比例 — 把 captureRatio 從 0.8 改成 0.5，看準確率變化。
練習 4新增一張統計卡片 — 例如「最近 20 次的平均準確率」。
練習 5Mode 2 雛形 — 把九宮格隨機顯示 1～9 的數字（先不做辨識）。
練習 6新增一個熱鍵 — 例如按 R 隨機切換干擾。

這些練習都「只改一小段」就能完成 — 練習的不是寫程式，是「能不能精準地請 AI 幫你改一小段」。

給 AI 工具的閱讀地圖

打開 index.html 後，請搜尋 SECTION: 這個字串。整份程式都用以下標記分區：

SECTION:STATE        全域變數、DOM、調色盤
SECTION:INTERFERENCE 干擾預設總表
SECTION:STORAGE      localStorage 存取
SECTION:TRUTH        Ground Truth 產生
SECTION:RENDER       Canvas 繪製
SECTION:DETECT       pixel → 顏色
SECTION:ANALYZE      單格分析 + 綜合加權
SECTION:SCAN         掃描整個九宮格
SECTION:LOGS         三種紀錄表格渲染
SECTION:CHARTS       分析圖表
SECTION:UI           演算法切換、主題、縮放
SECTION:HOTKEYS      鍵盤熱鍵
SECTION:TIMERS       自動產生／自動掃描
SECTION:EXPORT       CSV / JSON 匯出
SECTION:INIT         啟動流程

每個 SECTION 開頭都有一段給學生看的中文註解，告訴你「這一區在做什麼、為什麼這樣設計、想改什麼要動哪裡」。
