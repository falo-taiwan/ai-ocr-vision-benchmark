## 這份指南要幹嘛

如果你是學生，正在用 Gemini 免費版 / Pro、Gemini Canvas、Canva 之類的 vibe-coding 工具
學寫程式 — 這份指南就是寫給你看的。

它的目標**不是**教你怎麼從零寫程式，而是教你：

> 拿到一份別人寫好的程式，怎麼丟給 AI、怎麼問問題、怎麼請它幫你改。

這份範例（`index.html`）就是給你練習用的「素材」。

## 你需要先知道的 5 件事

### 1. 程式就是一份檔案

打開 `index.html`，瀏覽器就會跑。沒有後端、沒有 API key、沒有 npm、沒有 React。
這是刻意這樣設計的 — 因為你越能把整份東西當「一個檔案」看待，越容易丟給 AI。

### 2. 程式裡面被切成很多區（SECTION）

整份程式裡面到處都有這種標記：

```
/* === SECTION:TRUTH === Ground Truth 產生 ... */
```

當你想找「答案是怎麼產生的」，就搜尋 `SECTION:TRUTH` 直接跳過去。
當你想找「按鍵盤會怎樣」，就搜尋 `SECTION:HOTKEYS`。

請 AI 解釋程式時，你可以說：

> 「請只解釋 SECTION:ANALYZE 那一段在做什麼，給我國中生版本。」

這樣 AI 不會回你一大堆雜訊。

### 3. 紀錄（Log）才是 AI 真正在意的

這個 benchmark 會把每一次「掃描」都寫進三種紀錄：

- **truthLog** — 系統知道的正確答案
- **captureLog** — 掃描結果（用某種演算法）
- **weightedDecisionLog** — 用綜合加權算出來的決策

按「**匯出 JSON**」可以把這三份紀錄打包下載。然後你可以這樣請 AI 分析：

> 「我貼一份 JSON 給你，請告訴我這次測試中，哪個演算法在哪種干擾下最容易出錯。」

### 4. 「Ground Truth」是這個範例最重要的觀念

很多人以為「正確答案」是「畫面上看起來最明顯的東西」。錯。

正確答案是「**任務定義決定的東西**」。

舉例：

- 如果題目是「測試底圖辨識在強遮罩下會不會壞」→ 答案永遠是底圖色
- 如果題目是「測試哪個顏色佔比較多」→ 答案是面積較大的顏色

這個工具兩種情境都有，因為實際做 AI 系統時，你會經常遇到同一張圖片，
**不同任務就有不同的正確答案**。

### 5. 「演算法」不是越多越好，是「會不會打架」

工具裡有 4 個演算法選項：

| 選項 | 看哪裡 |
|---|---|
| 中央區域平均 | 只看格子中間那塊 |
| 整格平均 RGB | 看整個格子 |
| 主要顏色投票 | 數每個 pixel 屬於哪種顏色 |
| 綜合加權 | 上面三種加權平均 |

工具裡有一個觀念很重要：**單一演算法是綜合加權的特例**。
中央區域平均 = 100/0/0、整格平均 = 0/100/0、主要顏色投票 = 0/0/100。

所以「換演算法」其實只是換權重。請 AI 解釋這件事時，可以說：

> 「請解釋為什麼 v0.06 把單一演算法視為綜合加權的特例。這樣設計有什麼好處？」

## 操作 8 步驟

1. 用瀏覽器打開 `index.html`
2. 按「**產生一次**」（或按 `G`）
3. 按「**掃描一次**」（或按 `C`）
4. 試試切換演算法 → 重新掃描
5. 在干擾下拉選單選一個（例如「白霧 68%」）→ 重新掃描
6. 按 `A`（或點「分析圖表」）看圖表
7. 按「**匯出 JSON**」
8. 把 JSON 或整份 HTML 貼給 AI

## 把程式丟給 AI 的 Prompt Patterns

### Pattern A：請 AI 解釋

```
請假設我是國中生。
我會給你一份 HTML 檔案。
請只回答我問的問題，不要主動改程式。

第一個問題：請用 5 句話解釋這份程式在做什麼。
```

```
我會在後續訊息給你這份 HTML。
請先回答「我準備好了」，不要急著解釋。

[下一則訊息] [貼上 index.html 內容]
[下一則訊息] 請解釋 SECTION:DETECT 在做什麼。
```

### Pattern B：請 AI 改一小段

```
我想在這份 HTML 裡新增一種干擾，叫做 rainbowStripe。
規則：在每個格子上面畫 4 條斜向彩虹條紋，每條占格子寬度的 10%。

請：
1. 告訴我應該改哪一段（用 SECTION: 標記）
2. 給我可以直接貼回去的程式碼
3. 解釋你為什麼這樣寫
4. 提醒我這次修改可能會影響哪些其他功能
```

### Pattern C：請 AI 幫你看資料

```
我把這次測試的 JSON 貼給你。
請幫我做三件事：
1. 統計哪個演算法的整體準確率最高
2. 列出最常被誤判的「真值 → 誤判結果」前 3 名
3. 如果要改善，建議我下次測試應該調整什麼參數

[貼上匯出的 JSON]
```

### Pattern D：請 AI 教你拆程式

```
請假設我未來想把這份 index.html 拆成 3 個 .js 檔。
請先不要拆，只要回答：
1. 你會怎麼分？為什麼這樣分？
2. 哪一段最難拆？為什麼？
3. 拆完之後我要怎麼證明它沒壞？
```

### Pattern E：請 AI 做一個新 mode

```
這份工具目前只有 Mode 1（顏色九宮格）。
請幫我設計 Mode 2 數字九宮格的最小可運作版本：

需求：
- 每格隨機顯示 1～9 的數字（純白底，黑色數字）
- 系統知道正確答案（truthLog 要記錄）
- 先不做辨識，只做產生與顯示
- 不要動到 Mode 1 的程式

請先給我計畫，再給我程式碼。
```

## 6 個練習任務

### 練習 1：改顏色 ⭐
找到 `SECTION:STATE` 裡的 `palette`，把 9 個顏色換掉。
> AI prompt：「請把這份 HTML 的 palette 換成柔和粉色系，9 種顏色都要好區分。」

### 練習 2：新增干擾 ⭐⭐
複製一個 `interferencePresets` 裡的條目，改參數。
> AI prompt：「請新增一種叫 mosaic 的干擾，把畫面切成 8x8 馬賽克，比例 50%。」

### 練習 3：改 ROI 比例 ⭐
找到 `captureRatio = 0.8`，改成 `0.5`，重新掃描。
> 觀察重點：準確率會怎麼變？為什麼？

### 練習 4：新增統計卡片 ⭐⭐
在控制面板那一欄加一張「最近 20 次平均準確率」的卡片。
> AI prompt：「請在 controls 區塊新增一個 stat-card，顯示最近 20 筆 captureLog 的平均準確率。」

### 練習 5：Mode 2 雛形 ⭐⭐⭐
照 Pattern E 那段 prompt，做一個只會顯示數字的 Mode 2。
> 注意：先不要做辨識，只做「產生 + 顯示 + truthLog」。

### 練習 6：新增熱鍵 ⭐
找到 `SECTION:HOTKEYS` 的 `handleHotkey`，新增一個 `R` 鍵。
> AI prompt：「請新增一個熱鍵 R：按下時隨機切換干擾下拉選單到任一種干擾。」

## 給自己的小提醒

1. **AI 寫的程式不一定能跑** — 一定要貼回去試一次。
2. **AI 改完一段，請它「總結這次改了哪些檔案、哪幾行」** — 不然你會找不到自己改過什麼。
3. **不要一次叫 AI 改 5 件事** — 每次只改 1 件，貼回去確認能跑，再改下一件。
4. **看不懂 AI 寫的東西就直接說「我看不懂這段，請解釋」** — 不要假裝看得懂。
5. **學會看註解** — 這份程式的中文註解是寫給你看的，不是寫給機器看的。
