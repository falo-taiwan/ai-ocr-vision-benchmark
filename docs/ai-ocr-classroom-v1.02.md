# AI OCR Classroom v-b.203 說明

## 這是什麼

這是一個單檔 HTML 的教室座位 SVG 工具，用來教初學者理解：

- SVG 畫面配置
- 老師圖形座標系
- 學生固定座位座標系
- 男女性別標記
- 工程模式註解圖層
- JSON / Excel 匯入匯出閉環
- 靜態頁面的治理門檻

## v-b.203 重點

- 預設畫面去識別化，不直接顯示學生姓名與性別。
- 既有範例資料來自老師提供的 JSON，程式碼只保存 AES-GCM 密文，不保存姓名明文。
- 輸入密碼後才會解密載入人物、老師位置、座位與色卡資料。
- 老師仍是 `Force`，老師小圓點固定綠色，不受男女選項影響。
- 「案例：載入範例團體」會把受保護資料全覆蓋載入目前資料庫。
- 程式碼內加入大量 `Falo X Force Cheng` 註解，方便同學把 HTML 當成教學版原始碼閱讀。

## 範例資料治理

範例資料載入密碼是 `20260522`。

在 GitHub Pages 這種靜態網站中，這個密碼門檻是教學級治理示範，不是後端權限控管。程式使用瀏覽器 Web Crypto 做密碼雜湊與 AES-GCM 解密，目的是讓學生理解：

- 範例資料不應該無條件載入
- 人名與座位資料要有治理邊界
- 載入資料是一個授權動作
- 載入或匯入後會全覆蓋目前資料
- 預設畫面應先去識別化

## 資料表設計

匯出 JSON / Excel 時，資料會以資料庫概念整理：

| Sheet | 用途 |
|---|---|
| people | 人物資料：姓名、角色、性別、色卡、座位或自由座標 |
| seats | 座位資料：固定桌位與座位座標 |
| colors | 色卡資料：OCR 用 RGB 色彩基準 |
| rules | 工程規則：角色邊界、座標系、匯入規則 |
| views | 視角定義：老師視角與學生視角 |

## 座標系

老師使用圖形座標系：

- 記錄 `x / y`
- 只能在老師區域內移動
- 不吸附學生座位

學生使用座位絕對座標系：

- 記錄 `seatId`
- 桌子與座位不可移動
- 拖到已有人的座位時，兩人交換位置

## 工程模式

工程模式以學生視角作為標準座標。點選案件後，系統會自動切到學生視角，再在座位 SVG 上浮出註解圖層。

這樣可以避免老師列印視角與工程標定視角混淆。
