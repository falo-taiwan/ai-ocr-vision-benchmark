AI OCR Classroom v-b.203 說明

單檔 HTML 教室座位 SVG 工具，用一個範例教 SVG、座標系、資料庫表、治理門檻與匯入匯出閉環。

v-b.203 重點

  • 預設畫面去識別化，不直接顯示學生姓名與性別。
  • 既有範例資料來自老師提供的 JSON，程式碼只保存 AES-GCM 密文，不保存姓名明文。
  • 輸入密碼後才會解密載入人物、老師位置、座位與色卡資料。
  • 「案例：載入範例團體」會把受保護資料全覆蓋載入目前資料庫。
  • 程式碼內加入大量 Falo X Force Cheng 註解，方便同學把 HTML 當成教學版原始碼閱讀。
  • 老師 Force 小圓點固定綠色，不受男女選項影響。

治理邊界

GitHub Pages 是靜態網站，因此密碼門檻是教學級治理示範，不是後端權限控管。這裡用 Web Crypto 做密碼雜湊與 AES-GCM 解密，說明：範例資料不應無條件載入，預設畫面應先去識別化。

資料表設計

Sheet用途
people人物資料：姓名、角色、性別、色卡、座位或自由座標
seats座位資料：固定桌位與座位座標
colors色卡資料：OCR 用 RGB 色彩基準
rules工程規則：角色邊界、座標系、匯入規則
views視角定義：老師視角與學生視角

座標系

老師使用圖形座標系，記錄 x / y，只能在老師區域內移動。學生使用座位絕對座標系，記錄 seatId，拖到已有人的座位時會交換。

工程模式

工程模式以學生視角作為標準座標。點選案件後會自動切到學生視角，再在座位 SVG 上浮出註解圖層。
